## 1. MS Office(Word·Excel·PowerPoint) 파일 분석 지원

PDF·HWP에 이어 **MS Office 문서**까지 분석 대상 확장. 직접 렌더링이 불가하므로 HWP와 동일하게 **Office COM 자동화로 임시 PDF 변환 후 기존 분석 파이프라인 재사용**.

- **`src/convert_office.ps1`(신규)**: 확장자로 분기하여 각 앱 COM으로 PDF 변환.
  - Word(`.doc`/`.docx`): `Documents.Open(…, ReadOnly)` → `Document.ExportAsFixedFormat($out, 17)` (17=wdExportFormatPDF)
  - Excel(`.xls`/`.xlsx`): `Workbooks.Open(…, ReadOnly)` → `Workbook.ExportAsFixedFormat(0, $out)` (0=xlTypePDF)
  - PowerPoint(`.ppt`/`.pptx`): `Presentations.Open(…, WithWindow=msoFalse)` → `Presentation.SaveAs($out, 32)` (32=ppSaveAsPDF). **PowerPoint.Application.Visible는 $false 설정 불가**라 창 없이 연다.
  - ⚠️ 이 스크립트는 **ASCII 전용**(한글 리터럴 없음)이라 `convert_hwp.ps1`과 달리 **UTF-8 BOM 불필요**. 오류는 영어로 출력하고 main.js가 한글 접두어를 붙임.
- **`main.js`**: `office:convertToPdf` IPC 추가. 동일 앱 단일 인스턴스 충돌 방지를 위해 **순차 큐(`officeQueue`)**로 처리. 파일 다이얼로그 필터에 Word/Excel/PowerPoint 추가.
- **`preload.js`**: `convertOfficeToPdf` 노출.
- **`src/index.html`**: `OFFICE_RE`(`docx?·xlsx?·pptx?`)·`CONVERT_RE`(HWP+Office 통합) 추가. `prepareFiles()`가 Office를 분석 전 PDF 변환. 파일 선택·드래그&드롭 모두 허용, 헤더/안내 문구·저장 파일명 처리 확장.

### 검증 (end-to-end, 실제 변환 PDF 생성 확인)
- Word `.docx` → PDF (21.9 KB) ✅
- Excel `.xlsx` → PDF (20.0 KB) ✅
- PowerPoint `.pptx` → PDF (5.7 KB) ✅
- ⚠️ 함정: **좀비 Office 프로세스가 새 COM 자동화를 막는다.** 테스트 중 멈추면 `Get-Process WINWORD,EXCEL,POWERPNT | Stop-Process -Force` 후 재시도. (실제 변환 경로 `Open→ExportAsFixedFormat`은 정상)
- 전제: 해당 PC에 MS Office 설치 필요(이 환경엔 Word·Excel·PowerPoint 모두 설치 확인).

## 2. 스크롤해도 분석결과·처리옵션 항상 표시 (메인 스티키 패널 수정)

- **근본 원인**: `.container { overflow: hidden }`이 자식의 `position: sticky`를 무력화(스크롤 컨테이너 생성)하여 스티키 패널이 고정되지 않음.
  → **`overflow: clip`**으로 변경. 모서리 클리핑은 유지하되 스크롤 컨테이너를 만들지 않아 스티키 패널이 뷰포트 상단에 정상 고정.
- 스티키 패널 상단에 **분석결과 요약 pill**(전체·컬러·흑백·컬러비율) 추가(`.sticky-stats`/`.ss-pill`). 기존엔 처리옵션만 따라왔고 분석결과 카드는 스크롤되어 사라졌음. `updateStickyStats()`로 본문 통계와 항상 동기화(`syncSidebarPanel`·`updateSelectedCount`에서 호출).

## 3. 페이지 범위 목록은 스크롤 시 비고정

- `#rangeSummary`(컬러/흑백 페이지 범위 목록)를 **스티키 패널 밖**으로 이동. 분석 결과 상단에 한 번만 표시되고 스크롤하면 사라짐. 스티키 영역에는 요약 pill + 선택 컨트롤 + 처리옵션만 유지.

## 4. 사이드바 상단 패널도 스크롤 고정

- `#sbPanel`에 `position: sticky; top: 0; z-index: 10`(+구분 그림자) 적용. `#thumbSidebar`가 자체 스크롤 컨테이너(`overflow-y: scroll`)라 **사이드바 안에서 썸네일을 스크롤해도** 분석결과·처리옵션 패널이 상단에 고정.

## 5. 메인 창 컬러 페이지 숫자 초록색 표기

- 메인 통계 카드 `#colorPages` 숫자에 사이드바와 동일한 초록색(`#30d158`) 적용.
