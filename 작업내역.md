- **Shift+클릭**: 마지막 클릭 기준으로 범위 전체 선택

### 1-4. JPEG 흑백변환 → 프린터 칼라 오인식 수정
| | 기존 | 수정 후 |
|---|---|---|
| 인코딩 | JPEG (3채널 YCbCr) | FlateDecode (1채널 raw) |
| ColorSpace | `DeviceRGB` ❌ | `DeviceGray` ✅ |
| 프린터 인식 | 칼라 페이지 | 흑백 페이지 |

- `canvas.toBlob('image/jpeg')`은 R=G=B 픽셀이어도 항상 3채널 JPEG 생성
- 픽셀을 1채널로 추출 → `pako deflate` → `FlateDecode + DeviceGray`로 저장
- PNG Sub 예측 필터(`Predictor 15, Colors 1`) 적용으로 압축률 유지

## 3. 버그 수정 이력

### 3-1. 파일 선택 버튼 / 드래그 미작동 (3개 원인)

| # | 에러 | 원인 | 수정 |
|---|------|------|------|
| 1 | `module not found: fs` | Electron 샌드박스 모드에서 Node.js 모듈 차단 | `webPreferences`에 `sandbox: false` 추가 |
| 2 | `SyntaxError: await is only valid in async` | `exportToBusinessMgmt()`가 일반 함수인데 `await` 사용 | `async function`으로 변경 |
| 3 | 드래그 시 창이 파일 경로로 이동 | Electron 기본 동작 | `document` 레벨 `dragover/drop` 기본 동작 차단 |

### 3-2. PDF.js 워커 asar 로드 실패
- **원인**: 패키지 내 `app.asar` 안의 파일은 Web Worker로 직접 로드 불가
- **수정**: preload의 `fs.readFileSync`로 워커 파일 내용 읽기 → **Blob URL** 생성 → `workerSrc`에 적용

### 3-3. 대용량 PDF IPC 전송 실패
- **원인**: 파일 전체를 JSON 배열로 IPC 전달 → 50MB+ PDF에서 느리고 불안정
- **수정**: main은 **파일 경로만 반환**, preload의 `readFile(path)`로 `ArrayBuffer` 직접 반환

### 3-4. 내보내기 후 업무 관리 앱에 견적서 미표시
- **원인**: `shell.openPath()`로 시스템 브라우저에서 열면 Electron localStorage와 분리됨
- **수정**: `BrowserWindow.loadFile()`로 Electron 창에서 열기 → 동일 세션 localStorage 공유
- 추가: 이미 열려 있으면 새 창 대신 기존 창 포커스

## 5. 주요 설정값

### main.js webPreferences
```js
webPreferences: {
  preload: path.join(__dirname, 'preload.js'),
  contextIsolation: true,
  nodeIntegration: false,
  sandbox: false,   // preload에서 fs/path 사용 허용
}
```

### 줌 단축키 (main.js)
```js
win.webContents.on('before-input-event', (event, input) => {
  if (!input.control || input.type !== 'keyDown') return;
  if (input.key === '=' || input.key === '+') wc.setZoomLevel(wc.getZoomLevel() + 0.5);
  else if (input.key === '-')                  wc.setZoomLevel(wc.getZoomLevel() - 0.5);
  else if (input.key === '0')                  wc.setZoomLevel(0);
});
```

### IPC 채널 목록
| 채널 | 방향 | 역할 |
|------|------|------|
| `dialog:openFile` | renderer → main | 파일 열기 다이얼로그 (경로 반환) |
| `dialog:saveFile` | renderer → main | 저장 다이얼로그 + 파일 쓰기 |
| `shell:openBizApp` | renderer → main | 업무 관리 앱 BrowserWindow로 열기 |

### preload.js 노출 API
| 함수 | 역할 |
|------|------|
| `openFile()` | 파일 열기 다이얼로그 IPC 호출 |
| `readFile(path)` | Node.js fs로 파일 읽기 → ArrayBuffer 반환 |
| `saveFile(opts)` | 저장 다이얼로그 IPC 호출 |
| `openBizApp()` | 업무 관리 앱 열기 IPC 호출 |
| `getWorkerContent()` | PDF.js 워커 파일 내용 반환 (Blob URL용) |
