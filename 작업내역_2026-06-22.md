# 작업 내역 — 2026-06-22

PDF 분석기(Electron) UI/기능 개선. 모든 변경은 `src/index.html` 단일 파일에 반영했고,
포터블 앱(`dist/win-unpacked/resources/app/src/index.html`)에 동기화 완료.

---

## 1. 멀티탭 순서 마우스로 조절

- 각 탭(`.tab-item`)에 `draggable` 부여 + `dragstart/over/leave/drop` 핸들러 추가.
- 탭을 끌어 다른 탭 위에 놓으면 그 앞으로 이동 — `moveTabBefore(srcId, targetId)`가
  `tabs` Map을 새 순서로 재구성. 이 순서가 합치기 기본 순서에도 반영됨.
- 드래그 중 탭은 반투명(`.dragging`), 드롭 위치는 파란 막대(`.drag-over`) 표시.

## 2. 파일 합치기 순서 수정 모달

- `🔗 파일 합치기` 클릭 시 바로 병합하지 않고 **순서 조정 모달**(`openMergeDialog`) 표시.
- 목록 항목을 드래그하거나 ▲▼ 버튼으로 순서 변경 후 `🔗 합치기` → 그 순서대로 병합(`performMerge`).
- 취소/바깥 클릭으로 닫기.

## 3. 합본 파일명 `_합본`

- 기존 `합본_N개파일.pdf` → **맨 위 파일명 기준 `<원본이름>_합본.pdf`**.
  예: `견적서.pdf`가 맨 위면 `견적서_합본.pdf`.

## 4. 합본 파일 챕터(파일별) 구분

- 합치기 시 각 원본의 페이지 범위(`{name, start, count}`)를 `chapters`로 기록 →
  분석 완료 후 각 페이지에 출처 파일명(`r.chapter`) 태깅.
- **메인 그리드**: 파일 첫 페이지 앞에 전체 폭 헤더 `[파일 N] 📄 파일명 ··· N페이지`
  (`makeChapterDivider`, `chapterRunLength`).
- **사이드바**: 같은 위치에 번호 배지 + 파일명 구분줄(`.sb-chapter`).
- 페이지 태그 기준이라 삭제·회전·재정렬해도 구분이 따라감. 단일 파일 탭은 표시 안 함.

## 5. 사이드바 분석·처리옵션 패널 개선

- 사이드바 상단 패딩 제거 → 패널(`#sbPanel`)을 **맨 위에 밀착**(아래 모서리만 둥글게).
- 패널 헤더에 **iOS 스타일 토글 스위치**(`#sbPanelToggle`) 추가 → 본문(`.sbp-body`) 열기/닫기.
  상태는 `localStorage('sbPanelOpen')`에 저장되어 재실행 시 유지.

## 6. 사이드바 결과 새로고침 버튼

- 처리옵션 섹션에 `🔄 결과 새로고침`(`#sb-refreshBtn`) 추가 → 본문 `refreshResults()` 호출.
- 페이지 선택 시(`selectedPages.size > 0`)만 표시, `syncSidebarPanel()`에서 동기화.

## 7. 다운로드 버튼 우클릭 → 원본 저장

- 본문·사이드바 `📥 다운로드` 버튼 **우클릭 시 적용 여부와 무관하게 원본 PDF 즉시 저장**
  (`downloadOriginal`, 파일명 `원본이름.pdf`).
- 좌클릭은 기존대로 적용본 다운로드(적용 전이면 안내 메시지에 우클릭 안내 추가).
- 비활성 버튼은 우클릭 이벤트가 발생하지 않으므로, **PDF가 열려 있으면 버튼을 활성 유지**하되
  적용본이 없으면 흐리게(`.btn-dim`, 55% 불투명) 표시. 처리 중에만 완전 비활성.
- 두 버튼에 "좌클릭: 적용본 다운로드 · 우클릭: 원본 다운로드" 툴팁 추가.

## 8. 분석된 문서에 파일 추가 시 자동 챕터 합본

- **기존**: 파일을 가져올 때마다 항상 새 탭으로 따로 열림.
- **변경**: 이미 분석이 끝난 문서가 있는 상태에서 파일을 추가로 가져오면(파일 다이얼로그·드래그&드롭 모두),
  별도 탭 대신 **활성 분석본 뒤에 챕터로 이어붙여 하나의 PDF로 합침**. 결과는 단일 탭이라 다운로드 버튼으로
  하나의 파일로 저장 가능.
- 두 진입점이 모두 `startLoad()`를 거치므로 그 한 곳만 분기:
  - `isTabReady(t)`: 분석 완료(`status==='ready'` + 원본 바이트 + 결과 존재) 판정 헬퍼 추가.
  - 분석된 탭(활성 우선, 없으면 첫 ready 탭)이 있으면 `appendImportedFiles(files, baseTab)`로 위임.
- `appendImportedFiles`: pdf-lib로 베이스 페이지 복사(기존 `chapters` 보존, 없으면 전체를 한 챕터로) 후
  새 파일들을 차례로 챕터(`{name, start, count}`)로 추가 → 합본을 새 탭으로 분석하고 흡수된 베이스 탭은 닫음.
  기존 합본 챕터 렌더링 인프라(`makeChapterDivider`/사이드바 구분) 그대로 재사용.
- 합본 파일명: 이미 `_합본.pdf`면 유지, 아니면 `<베이스명>_합본.pdf`.
- 기존 `🔗 합치기`(여러 탭 수동 병합)는 그대로 유지.

### 빌드/배포

- 메모리 지침대로 **포터블(win-unpacked)만 갱신**. zip 재생성·설치형 빌드는 미실행.
- 확인 방법: `dist/win-unpacked/PDF 분석기.exe` 재실행.
- 8번 작업 반영 시 `npm run build`의 패키징은 성공했으나 마지막 exe 버전정보 각인(rcedit) 단계가
  `Unable to commit changes`로 실패 — exe 메타데이터만 건드리는 단계라 동작 무관. 부팅 스모크 테스트로
  정상 확인. `asar:false`이므로 향후엔 전체 재빌드보다 변경 파일을 `resources\app\`로 복사하는 편이 빠르고
  rcedit 오류도 회피됨.
