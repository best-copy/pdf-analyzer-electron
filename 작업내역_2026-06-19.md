## 1. 「적용」 / 「다운로드」 버튼 분리

기존 단일 `📥 적용 후 다운로드` 버튼을 두 개로 분리.

- **✅ 적용** (`applyChanges`): 수정사항(흑백변환 옵션·페이지 편집·페이지 선택)이 **추가되면 활성화**. 누르면 디스크에 바로 저장하지 않고 결과 PDF를 메모리(`processedPdfBytes`)에 생성.
- **📥 다운로드** (`downloadProcessed`): **적용이 끝나야 활성화**, **적용 진행 중에는 비활성화**. 메모리의 결과 PDF를 저장 다이얼로그로 저장.
- 적용 후 새 수정이 생기면 직전 결과를 무효화(`invalidateProcessed`)하여 다운로드를 다시 잠금 → 재적용 필요.
- 관련 상태: `processedPdfBytes`, `processedFileName`, `applying`. 탭 전환 시 결과 무효화.
- CSS: `.download-btn`(파란색), `.action-btn-group` 추가.

## 2. 문서 페이지 크기 표시 (파일 용량 → 실제 페이지 크기)

상단 정보줄에 파일 용량(MB) 대신 **PDF 페이지의 실제 물리 크기**를 표시.

- 분석 시 페이지별 크기(pt) 저장: `analyzePageColor` → `pageWpt` / `pageHpt` (회전 반영된 표시 크기).
- pt → mm 환산 후 표준 용지 판별(`detectPaperName`, ±3mm): A3/A4/A5/A6, B4/B5/B6, Letter/Legal/Tabloid + 가로/세로 방향.
- **혼합 문서 지원**(`updateFileInfo`): 크기별로 그룹화하여 표기.
  - 동일: `📐 210 × 297 mm (A4·세로)`
  - 혼합: `📐 혼합 — 210 × 297 mm (A4·세로) 1-3,5p · 297 × 420 mm (A3·가로) 4p`
- 빈 페이지는 `pageSize` 기준. 편집/undo/redo 후 `rerenderPages`에서 갱신.

## 3. 견적서 할인율(%) 항목 추가

- 견적 표에 **할인율(%)** 열 추가 (0~100, 품목별 입력).
- 합계 계산: `itemTotal = 수량 × 단가 × 부수 × (1 − 할인율%)` (`itemTotal` 헬퍼).
- 화면 표·소계/합계(`recalcQuote`)·인쇄/PDF(`buildQuoteHTML`)·파일명 금액에 모두 반영.
- 기본 품목·품목 추가 시 `discount: 0` 포함.

## 4. 실행취소(Undo) / 다시실행(Redo)

기존 "삭제 되돌리기"(단방향)를 **양방향 히스토리 시스템**으로 교체.

- **↩ 실행취소** / **↪ 다시실행** 버튼 분리 (가능할 때만 표시).
- 단축키: `Ctrl+Z` 실행취소 / `Ctrl+Y`·`Ctrl+Shift+Z` 다시실행.
- **삭제·삽입·회전·순서이동(드래그) 모두** 취소/재실행 지원.
- 구현: 편집 직전 페이지 순서(참조 배열) + 회전값 스냅샷 저장(`snapshotPages`/`pushHistory`), `undoEdit`/`redoEdit`/`restoreSnapshot`이 두 스택을 오감. 보관 한도 `HISTORY_LIMIT = 1000`(사실상 무제한). 새 편집 시 redo 스택 비움.
- 탭별 `undoStack`/`redoStack` 보관(`deletedPages` 제거).

## 5. HWP / HWPX 파일 분석 지원

PDF 전용이던 분석기를 한글 문서(HWP·HWPX)까지 확장. 직접 렌더링이 불가하므로 **한컴오피스 한글 COM 자동화로 임시 PDF 변환 후 기존 PDF 분석 파이프라인 재사용**.

- `src/convert_hwp.ps1`(신규): `HWPFrame.HwpObject` 로 `Open`(확장자 자동감지) → `FileSaveAsPdf` + `Format="PDF"`. **FileName만 주면 HWP로 저장되므로 Format 필수.**
- `main.js`: `hwp:convertToPdf` IPC 추가. 한글 단일 인스턴스라 **순차 큐**로 처리. 파일 다이얼로그 필터에 hwp/hwpx 추가.
- `preload.js`: `convertHwpToPdf`, 드롭 경로 취득용 `getPathForFile`(`webUtils`) 노출.
- `src/index.html`: `prepareFiles()`로 hwp/hwpx를 분석 전 PDF 변환. 파일 선택·드래그&드롭 모두 허용, 제목/헤더/저장 파일명 처리 확장.
- 검증: HWP·HWPX 모두 실제 `%PDF-1.6` 변환 성공.

## 8. HWP/HWPX 권한창 자동 "모두 허용"

한글 자동화 시 뜨는 보안 권한창(`접근 허용(Y)/모두 허용(N)/허용 안 함(A)/모두 안 함(C)`)을 자동 승인. `convert_hwp.ps1` 내 백그라운드 워처(Start-Job)가 처리.

- 단계별 디버깅으로 규명한 사실:
  - 권한창은 **바탕화면 등 실제 경로에서만** 뜨고 %TEMP%에선 안 뜸. `RegisterModule`은 보안모듈 DLL 미등록이라 무효.
  - 한글은 **WPF**(창 `HwndWrapper[hwp.exe;…]`, Win32 자식 0개) → `BM_CLICK` 불가.
  - **UI Automation**으로 버튼(`모두 허용 : ALT+N`) 탐지 가능하나 `InvokePattern.Invoke()`는 **무효(no-op)**.
  - **해결**: 버튼 화면 좌표(`TryGetClickablePoint`)에 **실제 mouse_event 클릭** 후 커서 원위치 복원. 우선순위 `모두 허용 > 접근 허용 > 확인/예`, 거부 계열 미클릭.
  - Add-Type의 UIAutomation 어셈블리는 **전체 경로**로 참조(`GetRuntimeDirectory()`+`WPF\…dll`).
- **검증 완료**: 바탕화면 HWP 변환 시 권한창 자동 감지 → `모두 허용` 1회 클릭으로 닫힘 → 변환 완료(수동 클릭 없음).

> ⚠️ `convert_hwp.ps1`은 한글 문자열 리터럴 포함으로 **반드시 UTF-8 BOM 저장**(PowerShell 5.1 CP949 오인 방지). Write/Edit 후 BOM 유지 확인 필요.
